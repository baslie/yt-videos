## Редакторские правила

Транскрибация — это сплошной поток устной речи из распознавания: повторы, оговорки,
вставки «эээ», шутки, отступления, обрывки фраз. Задача — **переработать её в связную статью,
сохранив смысл, факты, цифры и кейсы**, но убрав словесный мусор.

**Делать:**
- Сохранять все факты, числа, имена, кейсы, суммы, проценты — это ценность конспекта.
- Группировать поток речи в логические разделы (`h2`) и подразделы (`h3`).
- Переписывать разговорные фразы в чистый письменный русский (инфостиль): короткие
  предложения, без воды и канцелярита.
- Выделять `<strong>` ключевые мысли и термины (умеренно — 1–3 на абзац максимум).
- Продающие блоки (реклама курса, тарифы, призывы купить) — **приводить нейтрально**,
  как факт: «автор собрал опыт в курс, состав такой-то». Не пересказывать в тоне продажника.
- Цены, валюту, числа форматировать единообразно: `24 990 ₽`, `16 млн ₽`, `×20`, `73 млн`.

**Не делать:**
- Не выдумывать факты, которых нет в транскрибации.
- Не сохранять оговорки, мат, шутки-вставки, отступления «не обращайте внимания на…».
- Не оставлять разговорные конструкции («ну», «как бы», «короче», «вот»).
- Не дублировать одну мысль в нескольких блоках.

**Объём:** конспект плотнее оригинала — обычно 12–16 секций `h2`. Время чтения в meta-row
прикидывать примерно (≈200 слов/мин).

## Структура статьи

`{{ARTICLE_BODY}}` собирается из секций. Каждая секция:

```html
<!-- 1 -->
<h2 id="about"><span class="num">01</span>О чём это видео<a class="anchor" href="#about">#</a></h2>
<p>…</p>
```

Правила:
- `id` секции — короткий латинский slug (`about`, `myths`, `cases`, `price`, `summary`).
- `<span class="num">01</span>` — порядковый номер с ведущим нулём, по порядку.
- `<a class="anchor" href="#…">#</a>` — якорь, повторяет `id` секции.
- Каждый `id` из `h2` **обязан** присутствовать в TOC, и наоборот (на нём держится scroll-spy).
- Подразделы внутри секции — `<h3>` (без номера и без `id`, если не нужен отдельный пункт TOC).
- Перед продающей частью уместен разделитель `<hr class="div">`.

## Оглавление (TOC)

`{{TOC_ITEMS}}` — один и тот же набор `<li>` вставляется в шаблон дважды (десктоп `<ul>` и
мобильный `<ol>`), менять не нужно — просто подготовить список:

```html
      <li><a href="#about">О чём это видео</a></li>
      <li><a href="#myths">Три мифа об онлайн-курсах</a></li>
      <li><a href="#summary">Итог</a></li>
```

Тексты пунктов — короткие, человеческие (не дублируют дословно длинные `h2`).

## Hero

- `{{EYEBROW}}` — тип + тема: `Конспект вебинара · Яндекс.Директ`, `Конспект видео · GetCourse`.
- `{{H1}}` — заголовок статьи (можно чуть длиннее и точнее, чем в карточке).
- `{{LEDE}}` — 1–2 предложения, о чём видео и какая польза.
- `{{TITLE}}` (тег `<title>`) — заголовок + хвост: `… — конспект вебинара`.
- `{{META_DESCRIPTION}}` — 1 предложение с ключевыми темами для SEO.
- `{{META_ROW}}` — строки через `<span class="sep">·</span>`:

```html
        <span>Спикер: <b>Ирина Комлева</b>, директор комьюнити-проектов GetCourse</span>
        <span class="sep">·</span>
        <span>Источник: канал <b>GetCourse Медиа</b></span>
        <span class="sep">·</span>
        <span>~25 минут видео · 10 минут чтения</span>
```

Спикера/источник брать из транскрибации, если названы; иначе описательно
(`директолог с 10+ годами практики`).

- `{{FOOTER}}` — нейтральная подпись: «Это текстовый конспект … Транскрибация переработана
  в связную статью; смысл, факты, цифры и кейсы сохранены, продающий блок приведён нейтрально.
  <a href="https://www.youtube.com/watch?v=…">Оригинал на YouTube</a>.»

## Компоненты тела статьи

### Абзац и базовый текст
```html
<p>Обычный абзац. Ключевые мысли — <strong>через strong</strong>.</p>
```

### callout — крупная выделенная мысль
Для главных тезисов, выводов, «вопросов на подумать». Первый `<p>` крупный, второй — пояснение.
```html
<div class="callout">
  <span class="k">Главная мысль</span>
  <p>Крупный тезис, который должен запомниться.</p>
  <p>Необязательное пояснение помельче.</p>
</div>
```

### note — врезка с примером/оговоркой
Для примеров, доказательств, второстепенных уточнений.
```html
<div class="note">Пример: продажи шли с возраста 45–54 по статистике Яндекса, но отдел продаж
  говорит, что таких людей нет — значит <b>реальный возраст другой</b>.</div>
```

### Списки: clean / check / cross
- `ul.clean` — нейтральный маркированный список (точки-акценты).
- `ul.check` — список «подходит / плюсы» (зелёные галочки).
- `ul.cross` — список «не подходит / минусы» (красные крестики).
```html
<ul class="clean"><li>пункт;</li><li>пункт.</li></ul>
<ul class="check"><li>кому подходит;</li></ul>
<ul class="cross"><li>кому не подходит.</li></ul>
```

### cases — карточки кейсов/результатов людей
Имя + подпись (точка А / роль) + крупное число-результат + описание.
```html
<div class="cases">
  <div class="case">
    <div class="case-head">
      <div class="case-name">Арсений Верещагин <span>с нуля</span></div>
      <div class="case-num">200 000 ₽/мес</div>
    </div>
    <p>Вышел на доход 200 тыс. ₽ в месяц, ведёт 5 клиентов.</p>
  </div>
</div>
```

### roles — развёрнутые карточки «вариантов/ролей»
Когда сравниваются несколько ролей/путей/тарифов с разбором «подходит / не подходит».
`.money` — инлайн-плашка для суммы.
```html
<div class="roles">
  <div class="role">
    <div class="role-top">
      <h3>Эксперт</h3>
      <div class="tag">Накопил знания и практику, готов передавать ученикам</div>
    </div>
    <div class="role-body">
      <p>Описание. Доход = <span class="money">кол-во учеников × средний чек</span>.</p>
      <p class="lbl">Подходит, если</p>
      <ul class="check"><li>…;</li></ul>
      <p class="lbl">Не подходит, если</p>
      <ul class="cross"><li>…;</li></ul>
      <p class="lbl">Пример</p>
      <p>Полина Рыбакова вышла на <span class="money">18 млн ₽</span>.</p>
    </div>
  </div>
</div>
```

### mech — «как это ломается» (тег + проблема → результат)
Для разбора механики/ошибок: что сделали и к чему привело. `.res` — итог со стрелкой.
```html
<div class="mech">
  <div class="mech-item">
    <div class="mech-top"><span class="tag">Объявления</span>Оставить только «дешёвое» объявление</div>
    <div class="mech-body">
      <p>Что сделали и почему казалось логичным.</p>
      <p>Что произошло на самом деле.</p>
      <p class="res">Лиды стали в 2 раза дороже.</p>
    </div>
  </div>
</div>
```

### steps — нумерованные шаги / итоговые тезисы
Авто-нумерация. `<b>` — заголовок шага, `<span>` — пояснение.
```html
<ol class="steps">
  <li><b>Реклама показывается всем</b><span>Ключи — лишь стартовая подсказка.</span></li>
  <li><b>Объявление создаёт аудиторию</b><span>Хотите другую — делайте другое объявление.</span></li>
</ol>
```

### price — таблица тарифов
```html
<table class="price">
  <thead><tr><th>Тариф</th><th>Что входит</th><th>Цена</th></tr></thead>
  <tbody>
    <tr>
      <td><b>С чатом</b></td>
      <td>Записи уроков и чат — бессрочно.</td>
      <td class="amt">24 990 ₽<small>≈ 2 600 ₽/мес в рассрочку</small></td>
    </tr>
  </tbody>
</table>
```

### stats — лента крупных цифр (3 в ряд)
```html
<div class="stats">
  <div class="stat"><div class="big">×20</div><div class="cap">рост бюджета при неизменной окупаемости</div></div>
  <div class="stat"><div class="big">25 000 ₽/день</div><div class="cap">бюджет за месяц с нуля</div></div>
  <div class="stat"><div class="big">16 млн ₽</div><div class="cap">личный рекорд бюджета</div></div>
</div>
```

### Прочее
- `<hr class="div">` — крупный разделитель между смысловыми частями.
- `<h3>…</h3>` — подзаголовок внутри секции.

## Выбор компонента под смысл

| Что в транскрибации | Компонент |
|---|---|
| Главный тезис, вывод, «вопрос на подумать» | `callout` |
| Пример, доказательство, оговорка | `note` |
| Перечисление пунктов-акцентов | `ul.clean` |
| «Кому подходит / плюсы» и «кому нет / минусы» | `ul.check` / `ul.cross` |
| Истории людей с результатами (короткие) | `cases` |
| Сравнение ролей/путей с разбором подходит/нет | `roles` |
| Разбор ошибки: сделали X → получили Y | `mech` |
| Пошаговый план или финальные тезисы | `steps` |
| Тарифы, цены | `price` |
| Несколько ярких цифр-результатов | `stats` |
| Обычное повествование | `<p>` + `<strong>` |

Не перегружать: основа статьи — абзацы `<p>`. Компоненты — акценты, а не каждый абзац.
Если кейсов один-два и они длинные — лучше `note` или `<p>`, а не `cases`.
