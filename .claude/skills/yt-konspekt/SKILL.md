# yt-konspekt

Скилл собирает из сырой транскрибации видео статью-конспект для проекта **yt-videos**
(`C:\Users\Roman\Desktop\yt-videos`, публикуется на https://roman-purtow.ru/yt-videos/).
Работает в формате квиза: задаёт недостающие вопросы, затем перерабатывает текст в связную
статью по устоявшемуся шаблону и встраивает её в проект.

Каждый конспект — это папка `ГГГГ-ММ-ДД-тема/` с двумя файлами: исходной транскрибацией
(`source.md`) и собранной статьёй (`index.html`). Все страницы — чистая статика без сборки,
npm и внешних CDN; стили инлайнятся в один файл.

## Ресурсы скилла

- `assets/article-template.html` — полный каркас статьи (CSS + JS + topbar + тема + видео +
  плейсхолдеры). **Копировать его как основу `index.html` и заменять `{{ПЛЕЙСХОЛДЕРЫ}}`.**
  Сам CSS/JS не переписывать.
- `assets/source-template.md` — стандарт исходника: frontmatter + заголовок + смысловые
  подзаголовки. **Использовать как основу `source.md` (см. Шаг 3).**
- `references/components.md` — каталог всех смысловых блоков (callout, note, cases, mech,
  steps, price, stats, roles, списки check/cross/clean) с разметкой + редакторские правила
  переработки транскрибации. **Прочитать перед сборкой тела статьи.**

## Воркфлоу

### Шаг 1. Квиз — собрать недостающие вводные

Спросить у пользователя ТОЛЬКО то, что он ещё не дал. Не спрашивать лишнего.

1. **Путь к файлу транскрибации** (`.md`/`.txt`) — если не указан. Прочитать файл.
2. **Ссылка на YouTube** — если не дана. Видео по умолчанию **всегда** встраивается в статью,
   поэтому ссылка нужна обязательно; запросить её, если пользователь не выдал.
   Извлечь из ссылки `YOUTUBE_ID` (часть после `watch?v=` или `youtu.be/`).
   - Если пользователь явно говорит, что видео не нужно — собрать статью без блока (см. Шаг 4).
3. **Slug темы** для имени папки — предложить свой вариант (короткий, латиницей, через дефис,
   например `yandex-direct`, `getcourse`) и дать подтвердить/поправить.

**Дату НЕ спрашивать** — всегда брать сегодняшнюю (из контекста сессии, поле "Today's date").
Имя папки: `ГГГГ-ММ-ДД-<slug>` (например `2026-06-13-yandex-direct`).

Спрашивать через инструмент вопросов (квиз), а не сплошным текстом. Если чего-то не хватает —
один компактный набор вопросов, а не по одному.

### Шаг 2. Прочитать транскрибацию и спланировать структуру

- Прочитать файл целиком (большие файлы — постранично).
- Прочитать `references/components.md`.
- Наметить разделы статьи (`h2`) по логике видео: 12–16 секций. Под каждый раздел —
  короткий латинский `id` и пункт оглавления.

### Шаг 3. Создать папку и сохранить нормализованный исходник

- Создать `C:\Users\Roman\Desktop\yt-videos\ГГГГ-ММ-ДД-<slug>\`.
- Привести вход к **стандартизированному Markdown** по `assets/source-template.md` и сохранить
  как `source.md`. Вход может быть в любом виде — сплошная стена текста, `.txt`, скопированные
  субтитры (SRT/VTT), расшифровка с таймкодами; задача — единый опрятный формат.

**Заполнить frontmatter:**
- `title` — название видео; `source` — площадка/канал (например `YouTube · GetCourse Медиа`);
  `youtube` — полная ссылка; `date` — сегодняшняя `ГГГГ-ММ-ДД`; `speaker` — если известен,
  иначе пустая строка `""`.

**Нормализация текста (бережно — это архив, не статья):**
- Содержание и формулировки **сохранять как в оригинале** — не переписывать в инфостиль,
  не сокращать смысл (причёсывание стиля делает только `index.html`).
- Убрать таймкоды (`00:01:23`, `[00:01]`), номера строк, технические артефакты.
- Если вход — субтитры SRT/VTT: выбросить индексы и тайминги, склеить реплики в связный текст.
- Разбить сплошной поток на **абзацы** (пустая строка между) по смыслу — по 3–7 предложений.
- Расставить смысловые подзаголовки `##` по крупным темам видео (можно согласовать с разделами
  будущей статьи) — для навигации по архиву.
- Допустимо убрать явный мусор распознавания: дубли слов подряд, «эээ», оборванные склейки.
  Реальные слова и факты не выбрасывать.
- Не оставлять ни одного `{{…}}` из шаблона.

### Шаг 4. Собрать `index.html` из шаблона

- Скопировать `assets/article-template.html` → `<папка>/index.html`.
- Заполнить плейсхолдеры (детали разметки каждого — в `references/components.md`):
  - `{{TITLE}}`, `{{META_DESCRIPTION}}` — `<head>`.
  - `{{EYEBROW}}`, `{{H1}}`, `{{LEDE}}`, `{{META_ROW}}` — hero.
  - `{{YOUTUBE_ID}}` — id ролика (в двух местах: embed и ссылка). Используется и в `title`
    iframe через `{{H1}}` — заменить `{{H1}}` и там тоже.
  - `{{TOC_ITEMS}}` — пункты оглавления (вставляются в шаблон в двух местах одинаково).
  - `{{ARTICLE_BODY}}` — тело статьи: секции `h2` + контент с компонентами.
  - `{{FOOTER}}` — нейтральная подпись со ссылкой на оригинал.
- **Переработать транскрибацию в связный текст** по редакторским правилам из
  `references/components.md`: сохранить факты/цифры/кейсы, убрать словесный мусор, продающие
  блоки подать нейтрально, расставить смысловые компоненты по таблице выбора.
- **Если видео не нужно** — удалить из шаблона блоки `<div class="video">…</div>` и
  `<div class="source-row">…</div>`, а в футере не давать ссылку на YouTube.
- Проверить: каждый `id` секции `h2` есть в `{{TOC_ITEMS}}` (на этом держится scroll-spy);
  не осталось ни одного `{{…}}`.

### Шаг 5. Встроить конспект в проект

**Корневой `C:\Users\Roman\Desktop\yt-videos\index.html`** — добавить карточку первой в `.grid`
(новые сверху), перед карточкой `.soon`:
```html
    <a class="card" href="ГГГГ-ММ-ДД-<slug>/index.html">
      <div class="date">D месяца ГГГГ · Тема</div>
      <h2>Заголовок конспекта</h2>
      <p>1–2 предложения, о чём конспект.</p>
      <span class="go">Читать конспект →</span>
    </a>
```

**`C:\Users\Roman\Desktop\yt-videos\README.md`:**
- Добавить строку первой в таблицу «Конспекты»:
  `| ДД.ММ.ГГГГ | Название видео | [читать](https://roman-purtow.ru/yt-videos/ГГГГ-ММ-ДД-<slug>/) |`
- При необходимости отразить новую папку в блоке «Структура».

### Шаг 6. Коммит

Закоммитить изменения (без push) осмысленным сообщением в стиле репозитория, например:
`Добавлен конспект: <тема> (ГГГГ-ММ-ДД)`. Использовать скилл `committing-changes`, если доступен.

## Проверки качества

- Открыть собранный `index.html` нелишне — статика, можно глазами проверить отсутствие
  `{{…}}` и совпадение `id` ↔ TOC поиском по файлу.
- Заголовки секций пронумерованы по порядку (`01`, `02`, …).
- Факты, числа, имена и кейсы из транскрибации сохранены; продающий блок нейтрален.
- Тёмная/светлая тема, оглавление и видео не трогаются — они уже в шаблоне.
